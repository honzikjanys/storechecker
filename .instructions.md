# StoreChecker Development Instructions

## Project Overview

StoreChecker is a Python application that monitors store locations for major Czech retail chains (Tesco, Penny Market, Billa, Albert, Kaufland). It scrapes store data, detects changes, and sends email notifications.

## Key Technologies

- **Python 3.8+**
- **BeautifulSoup4** - Web scraping
- **Selenium** - JavaScript-heavy sites
- **APScheduler** - Scheduled tasks
- **SQLite** - Data storage
- **SMTP** - Email notifications

## Architecture

### Core Components

1. **Scrapers** (`src/scrapers/`)
   - Base class: `BaseScraper`
   - Retailer implementations: `TescoScraper`, `PennyScraper`, etc.
   - Outputs: List of `Store` objects

2. **Database** (`src/database.py`)
   - SQLite manager
   - Stores store data
   - Tracks changes and history

3. **Notifier** (`src/notifier.py`)
   - Sends HTML/text email reports
   - Formats changes and summaries

4. **Scheduler** (`src/scheduler.py`)
   - APScheduler wrapper
   - Manages weekly jobs

5. **Main Application** (`main.py`)
   - Orchestrates all components
   - CLI interface (--once, --schedule)

## Development Guide

### Adding a New Retailer

1. Create `src/scrapers/retailer_scraper.py`
2. Extend `BaseScraper`
3. Implement `scrape()` method
4. Add to `SCRAPERS` list in `main.py`

See `SCRAPER_GUIDE.md` for detailed instructions on updating selectors.

### Common Tasks

#### Debugging a Scraper

```bash
python -c "
from src.scrapers.tesco_scraper import TescoScraper
scraper = TescoScraper()
stores = scraper.scrape()
print(f'Found {len(stores)} stores')
for s in stores[:3]:
    print(s)
"
```

#### Testing Email

```bash
python -c "
import config
from src.notifier import EmailNotifier
notifier = EmailNotifier(
    config.SMTP_SERVER, config.SMTP_PORT,
    config.SMTP_USERNAME, config.SMTP_PASSWORD,
    config.EMAIL_FROM
)
notifier.send_report(
    config.EMAIL_RECIPIENTS,
    {'new': [], 'temporarily_closed': [], 'permanently_closed': [], 'reopened': []},
    []
)
"
```

#### Checking Database

```bash
sqlite3 data/stores.db
> SELECT COUNT(*) FROM stores;
> SELECT * FROM stores LIMIT 5;
> SELECT * FROM store_history ORDER BY change_date DESC LIMIT 10;
```

### Configuration

- Copy `config.example.py` to `config.py`
- Fill in SMTP credentials
- Set schedule (day and time)
- Add email recipients

### Running

```bash
# Test run (execute once)
python main.py --once

# Start scheduler (weekly)
python main.py --schedule

# Custom schedule
python -c "
from main import StoreChecker
app = StoreChecker()
app.scheduler.add_weekly_job(
    app.check_stores,
    day='wednesday',
    time='14:30'
)
app.start()
"
```

## Important Notes

### Placeholder Selectors

Current scrapers use placeholder CSS selectors. Each must be updated:

1. Visit retailer's store locator page
2. Inspect HTML with browser DevTools (F12)
3. Find actual CSS selectors
4. Update scraper code
5. Test with `python main.py --once`

See `SCRAPER_GUIDE.md` for detailed instructions.

### Email Configuration

For Gmail:
1. Enable 2-Factor Authentication
2. Create [App Password](https://myaccount.google.com/apppasswords)
3. Use app password (not regular password) in `config.py`

For other providers:
- Find their SMTP server address
- Use application-specific passwords if available

### Database Schema

#### stores table
- `store_id`: Unique identifier (TEXT)
- `retailer`: Retailer name (TEXT)
- `name`: Store name (TEXT)
- `city`: City name (TEXT)
- `address`: Store address (TEXT)
- `status`: open/temporarily_closed/permanently_closed (TEXT)
- `phone`: Phone number (TEXT, optional)
- `hours`: Operating hours (TEXT, optional)
- `scraped_at`: When scraped (TIMESTAMP)
- `updated_at`: Last update (TIMESTAMP)

#### store_history table
- Tracks all changes (new, status changes)
- Used for email notifications
- Queryable by time window

#### scrape_runs table
- Records each scrape execution
- Status and timestamp
- Store count per run

## Troubleshooting

### Scraper returns no stores

- Check HTML selectors match actual website
- Verify website doesn't require JavaScript (try Selenium)
- Check User-Agent isn't being blocked
- Look at logs: `tail data/storechecker.log`

### Email not sending

- Verify SMTP credentials in `config.py`
- Check firewall/network allows SMTP port (587)
- Check logs for SMTP errors
- Test with `python -m smtplib` if needed

### Database errors

Reset database:
```bash
rm data/stores.db
python main.py --once
```

### Scheduler not running

Check:
- Job is added: `python -c "from main import StoreChecker; app = StoreChecker(); print(app.scheduler.get_jobs())"`
- Output to logs: `tail -f data/storechecker.log`
- Correct day/time format in `config.py`
